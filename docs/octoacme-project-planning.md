# OctoAcme — Project Planning

## Purpose
Turn an approved initiative into an actionable plan and backlog for delivery.

## Objectives
- Break work into shippable increments
- Identify dependencies and risks
- Align timelines, releases, and responsibilities

## Activities
1. Kickoff meeting with stakeholders and delivery team
2. Create prioritised backlog with acceptance criteria (BA leads requirement authoring; UX Designer contributes UX criteria)
3. Estimate scope (T-shirt sizing or story points)
4. Define Definition of Done (DoD) — include UX sign-off, security scan pass, and support-readiness items
5. Identify dependencies and integration points — Security Champion flags security-sensitive integrations; Support Lead flags supportability concerns
6. Create release plan and milestone map

## Sprint / Iteration Planning
- Timebox planning to agreed sprint length
- Pull items that meet DoD and have clear acceptance criteria
- Ensure team capacity is respected

## Risk & Dependency Management
- Capture in Risk Register:
  - ID, Description, Impact, Probability, Owner, Mitigation
- Mark cross-team dependencies in the project board and escalate during weekly syncs
- **Security Champion** adds security risks during planning and flags new ones at each sprint
- **Support Lead** logs supportability gaps (e.g., missing runbooks, tooling limitations) as risks

## Cross-functional Involvement Checklist
- [ ] UX Designer has reviewed/approved designs for items in the upcoming sprint
- [ ] Security Champion has completed a threat model for significant new features
- [ ] Business Analyst has finalised acceptance criteria for all planned items
- [ ] Support Lead has confirmed no outstanding supportability blockers

## Planning Checklist
- [ ] Project kickoff held
- [ ] Backlog prioritised and estimated (BA has authored/reviewed acceptance criteria)
- [ ] UX Designer has provided designs for in-scope user-facing work
- [ ] Security Champion has completed threat modelling for high-risk features
- [ ] Support Lead has reviewed planned features for supportability
- [ ] Release timeline and milestones agreed
- [ ] Definition of Done documented (includes UX sign-off, security scan pass, support-readiness)
- [ ] Initial test plan / QA approach drafted
